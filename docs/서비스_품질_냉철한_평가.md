# 서비스 품질 평가

최종 업데이트: 2026년 6월 3일
작성 기준: 현재 `main` 브랜치 코드와 실제 서비스 목적

이 문서는 `cat_stock`이 "AI에게 붙여넣기 좋은 주식/코인 리서치 재료를 만든다"는 목적을 얼마나 달성하고 있는지 평가합니다. 예전 문서의 오래된 혹평과 이미 해결된 항목은 지우고, 현재 사용자 경험 기준으로 다시 정리했습니다.

## 결론

지금 서비스는 실제 사용 가능한 단계입니다.

다만 "완성된 투자 리서치 서비스"라기보다는 "AI 분석을 위한 데이터 수집/정리 도구"입니다. 이 기준에서는 충분히 가치가 있습니다. 반대로 앱이 직접 결론을 내리거나 기관 리포트 수준의 완성 분석을 자동 생성한다고 보면 아직 부족합니다.

| AI 칼럼 | Beta | 결과가 좋으면 매력적이지만 실패 이유/테스트/비용 관리가 아직 약함 |
| 배포/운영 | B | Railway 배포 구조는 단순하지만 외부 사이트 파싱 의존 리스크가 있음 |

## 좋아진 점

### 1. 주식 모드가 더 이상 뼈대 수준이 아님

이전에는 개별 종목 기본 정보, 테마 뉴스, CAT COIN 등이 미구현 또는 mock에 가까웠습니다. 현재는 다음 데이터가 실제로 연결되어 있습니다.

- 네이버 금융 기반 종목 기본 정보
- 네이버 금융 종목 뉴스 탭 헤드라인/언론사/시간/링크
- 네이버 리포트와 fnguide 리포트
- DART 공시, 재무, 대주주 지분율
- 네이버 등락률 상위 중 +5% 이상 상승 종목
- KIS 키가 있을 때 수급 금액/공매도/선물 수급 보강

### 2. TXT와 HTML의 역할이 분리됨

이 방향은 좋습니다.

- TXT: AI 프롬프트에 넣기 좋은 핵심 지표
- HTML 대시보드: 사람이 읽기 좋은 카드/표/차트/뉴스 링크

뉴스 전문을 그대로 복제하지 않고 원문 링크 중심으로 둔 것도 유지하는 편이 좋습니다.

### 3. CAT COIN은 공부 도구로 방향이 맞음

코인은 주식처럼 프롬프트 재료만 뽑기보다 대시보드로 흐름을 보는 게 더 맞습니다. 현재 구현도 이 방향으로 잡혀 있습니다.

## 아직 부족한 점

### 1. 데이터가 비었을 때 이유가 충분히 설명되지 않음

사용자는 "데이터 없음"이 오류인지, 장 전이라 없는 것인지, API 키가 없어서 빠진 것인지 구분하기 어렵습니다.

예:

- DART 당일 공시는 새벽에 없을 수 있음
- KIS 키가 없으면 수급 금액이 비어 있음
- 네이버 구조 변경이면 특정 표가 비어 있음
- OpenAI 키가 없으면 AI 칼럼이 없음

개선 방향:

- 결과 payload에 `source_status` 추가
- UI 경고를 "데이터 없음"이 아니라 "왜 없는지"까지 표시

### 2. AI 칼럼은 매력적이지만 아직 Beta

신문 스타일 칼럼은 사용자가 읽기 좋습니다. 다만 원자료가 빠져 있거나 API 호출이 실패하면 품질이 흔들립니다.

현재 리스크:

- 실패 이유가 `None`으로 사라짐
- OpenAI 호출 비용/속도/할당량 관리가 문서화되지 않음
- 칼럼이 사실을 과장할 수 있음
- 칼럼 기능 테스트가 부족함

서비스적으로는 "AI 해설"이 아니라 "AI 칼럼 초안"으로 표현하는 편이 안전합니다.

### 3. 테마 공부는 아직 얕음

테마 공부는 초보자가 키워드를 훑는 용도로는 괜찮습니다. 하지만 AI에게 깊은 투자 포인트를 물어보려면 데이터가 부족합니다.

부족한 것:

- 테마 종목별 시가총액
- 테마 종목별 PER/PBR
- 테마 관련 공시
- 글로벌 피어
- 대장주/후발주 구분

### 4. 속도는 더 좋아질 수 있음

코인 서비스는 병렬화가 잘 들어가 있습니다. 주식 서비스는 아직 순차 호출이 많아 종목 분석이 느릴 수 있습니다.

개선 우선순위:

1. DART/네이버/fnguide/KIS 병렬화
2. 같은 날짜/종목 반복 조회 캐시
3. AI 칼럼은 별도 버튼 또는 비동기 옵션 검토

### 5. 네이버 HTML 파싱 의존 리스크

서비스가 네이버 금융 HTML에 많이 의존합니다. 이건 개인용/교육용 도구로는 현실적인 선택이지만, 운영 안정성 면에서는 약점입니다.

대응:

- 주요 파서 fixture 테스트
- 페이지 구조 변경 시 스모크 체크
- 실패 섹션을 사용자에게 명확히 표시

## 사용자 관점에서의 현재 가치

### 주식 초보자에게 좋은 점

- 코스피/코스닥, 수급, 거래대금, 뉴스, 공시를 한 화면에서 볼 수 있음
- 종목 하나를 넣으면 기본 재료가 한 번에 모임
- AI에게 붙여넣기 좋은 텍스트가 나와서 질문을 시작하기 쉬움
- 대시보드 HTML로 숫자를 표와 카드 형태로 볼 수 있음

### 주식 초보자가 조심해야 할 점

- 출력은 정답이 아니라 재료입니다.
- "등락률이 높다"는 좋은 종목이라는 뜻이 아닙니다.
- "외국인이 샀다"는 무조건 상승한다는 뜻이 아닙니다.
- DART 공시/뉴스는 맥락을 직접 확인해야 합니다.
- AI 칼럼은 해설 초안이지 투자 권유가 아닙니다.

## 유지할 서비스 방향

이 앱은 투자 판단 자동화 서비스가 아니라 리서치 재료 정리 서비스로 가는 게 맞습니다.

유지하면 좋은 원칙:

- 프롬프트용 TXT는 짧고 구조적으로 유지
- 대시보드는 사람이 읽기 좋게 유지
- 뉴스는 헤드라인/링크 중심 유지
- AI 칼럼은 선택적 보조 기능으로 유지
- 데이터가 없을 때 조용히 숨기지 말고 이유를 보여주기
