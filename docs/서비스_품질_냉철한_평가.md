# 서비스 품질 냉철한 평가

작성일: 2026년 5월 31일  
작성 기준: 현재 develop 브랜치 코드 직접 분석

이 문서는 "AI에게 붙여넣기 좋은 양질의 주식 리서치 텍스트를 만들어준다"는 서비스 목적을 기준으로,
현재 cat_stock이 얼마나 그 목적을 달성하고 있는지 냉철하게 평가한다.

---

## 결론 먼저

**지금 상태에서 실제로 쓸 수 있는가? → 절반만 그렇다.**

시황 브리핑과 개별 종목 분석은 기본 뼈대가 갖춰져 있다.
그러나 데이터 품질 문제, 안정성 문제, AI 활용도 문제가 겹쳐 있어서
지금 당장 "기관 리서치 수준의 AI 분석 재료"를 만들어내지는 못한다.
테마 공부는 아직 뼈대 수준이다.

## 1. 데이터 품질 — 빠진 것, 틀릴 수 있는 것

### 치명적 공백

**공매도 잔고 비율 없음**
- 현재 코드: `short_balance_ratio: None` 하드코딩
- 영향: 개별 종목 분석의 "수급" 섹션에 공매도 비율이 항상 빈 칸으로 출력됨
- 공매도 잔고는 단기 급락 리스크 판단의 핵심 지표이므로 없으면 수급 분석이 절름발이
- KRX 공개 데이터로 수집 가능하지만 URL 구조 파악이 필요한 상태

**코스피200 선물 외국인/기관 순매수 없음**
- 현재 코드: `futures_foreign_net: None` 하드코딩
- 영향: 시황 브리핑에서 "외국인 코스피200 선물 순매수 데이터 없음"이 출력됨
- 선물 수급은 다음날 현물 방향을 예측하는 데 중요한 지표

### 수치 정확성 의심 구간

**외국인/기관 수급이 주 수 기준**
- 현재 코드: `_format_signed_shares()` → `+1,245,000주`
- 금액 기준(억 원)이 아니라 주 수 기준으로 출력됨
- AI에게 이 단위를 설명하지 않으면 "1,245,000억 원 순매수"로 잘못 해석할 수 있음
- 실제 금액 의미와 10배~100배 오차가 날 수 있어 분석 왜곡 위험

**업종별 등락 파싱 추정 기반**
- 현재 코드: 네이버 sise_group.naver에서 `cells[3]`을 등락률로 추정 파싱
- 컬럼 순서가 실제로 맞는지 검증된 적 없음
- 네이버가 테이블 구조를 바꾸면 전혀 다른 숫자가 등락률로 출력될 수 있음

**시간외 단일가 급등락 불완전**
- 등락률 파싱이 CSS 클래스 기반 부호 감지에 의존
- 장 마감 전에는 페이지에 데이터가 없을 수 있음

**테마 종목 데이터 빈약**
- 종목명, 현재가, 등락률만 수집
- 시가총액/PER/PBR 없음 → AI가 밸류에이션 맥락 없이 분석해야 함

## 2. 안정성 — 언제 조용히 깨지는가

### 크롤링 의존도 집중 리스크

현재 데이터 수집의 80% 이상이 네이버 증권 HTML 파싱에 의존한다.

- 네이버가 페이지 구조를 바꾸면 → 특정 섹션이 조용히 빈 칸으로 출력됨
- 네이버가 User-Agent 차단을 강화하면 → 전체 수집 실패
- 실제로 이미 일부 페이지는 인코딩 처리(cp949) 등 특수 처리가 필요한 상태

### 실패가 조용히 숨겨짐

현재 모든 수집 함수는 예외 발생 시 `None` 또는 빈 리스트를 반환한다.

```python
except Exception:
    return []
```

이 구조의 문제:
- 사용자가 생성 버튼을 눌렀을 때 50%의 데이터가 빠져 있어도 알 수 없음
- 텍스트에 "데이터 없음"이 다수 등장하면 AI가 맥락 없이 분석하게 됨
- 어느 수집기가 실패했는지 화면에서 피드백이 전혀 없음

### 속도 문제

개별 종목 분석 기준으로 현재 동작하는 수집 경로:
1. FinanceDataReader KRX 전체 종목 조회 (lru_cache 있음, 첫 호출만 느림)
2. 네이버 종목 메인 페이지 파싱
3. 네이버 frgn.naver 페이지 파싱 (investor flow + consensus + report rows)
4. 각 리포트 상세 페이지 최대 5개 순차 요청 (_enrich_report)
5. fnguide SVD_Report_Summary_Data
6. 네이버 뉴스 검색
7. DART corp code 조회
8. DART 공시 목록
9. DART 재무 데이터 4개 분기 순차 조회
10. DART 대주주 지분

→ 순차 I/O가 10개 이상. 빠른 환경에서도 15~30초 소요 예상.
생성 중 타임아웃이나 사용자 이탈 가능성이 높다.

## 3. AI 활용도 — 붙여넣어서 실제로 유용한가

### 현재 출력 텍스트의 구조적 문제

지금 생성되는 텍스트의 형태:

```
[시황 브리핑 데이터 - 2026-05-31]

■ 한국 지수
코스피 지수 2,650 | 등락률 +0.8% | 거래대금 12.4조 | 전일대비 +21.2pt
...

■ 글로벌/매크로
다우 +0.36%
S&P500 +0.74%
...
```

이 텍스트를 AI에게 붙여넣으면 AI는 무엇을 해야 할지 모른다.
분석 지시 프롬프트가 없기 때문에 사용자가 "이걸 분석해줘"라고 따로 입력해야 한다.

서비스 목적은 "AI가 분석하기 좋은 재료를 만들어주는 것"인데,
현재는 재료만 있고 어떻게 요리할지 방향이 없다.

### 단위·맥락 누락

- 수급 단위가 주 수인지 금액인지 텍스트에 명시 없음
- PER/PBR이 어느 기준인지 (TTM/Forward) 명시 없음
- 이동평균선 괴리율이 언제 기준인지 명시 없음
- 재무 데이터가 연간인지 분기인지 혼재될 수 있음

AI가 단위를 모르면 분석이 틀려진다.

### 테마 공부의 실질적 한계

현재 테마 공부가 제공하는 것:
- 종목명 + 현재가 + 등락률
- 뉴스 헤드라인 8개

이것만으로 AI가 "이 테마의 투자 포인트와 밸류체인을 설명해줘"를 제대로 답하기 어렵다.
최소한 시가총액, PER 수준은 있어야 테마의 규모와 밸류에이션 위치를 파악할 수 있다.

## 4. 우선순위별 보완 권고

### 즉시 해야 할 것 (서비스 신뢰도에 직결)

1. **수급 단위 명시**
   - 출력 텍스트에 "외국인 20일 수급: +1,245,000주 (주 수 기준, 금액 아님)" 처럼 단위 명기
   - 작업 난이도: 매우 낮음 (formatter 수정만 필요)

2. **업종별 등락 컬럼 검증**
   - 실제 앱 실행 후 네이버 sise_group.naver 페이지와 출력값 대조
   - 작업 난이도: 낮음 (1시간 이내)

3. **수집 실패 피드백**
   - 데이터 수집 중 실패한 섹션을 사용자에게 표시 ("⚠ 업종 데이터 수집 실패")
   - 작업 난이도: 낮음

### 빠르게 해야 할 것 (데이터 완결성)

4. **공매도 잔고 비율**
   - KRX 공개 API 또는 네이버 종목 페이지에서 수집 경로 확보
   - 작업 난이도: 중간 (수집 경로 탐색 + 파싱)

5. **테마 종목 시총/PER 추가**
   - 현재 theme_detail.naver에서 가져오는 데이터에 밸류에이션 컬럼 추가
   - 작업 난이도: 낮음~중간

6. **분석 프롬프트 자동 첨부 (선택 옵션으로)**
   - 텍스트 하단 또는 별도 영역에 "이 데이터를 바탕으로 [시황/종목/테마]를 분석해줘" 형태의 기본 프롬프트 추가
   - 사용자가 켜고 끌 수 있게 체크박스
   - 작업 난이도: 낮음, 임팩트 높음

### 중기에 해야 할 것 (안정성·속도)

7. **수집 병렬화**
   - 현재 10개 이상 순차 I/O를 `ThreadPoolExecutor`로 병렬화
   - 예상 개선: 30초 → 8~12초

8. **캐싱 도입**
   - `@st.cache_data(ttl=300)` 적용으로 같은 종목 반복 조회 시 크롤링 생략

9. **코스피200 선물 수급**
   - KRX 또는 네이버 파생상품 페이지에서 수집
   - 시황 브리핑의 핵심 누락 데이터

## 5. CAT COIN 현황

기획서는 완성도 있게 작성되어 있다.  
현재 코드 상태: 로고 클릭으로 모드 전환 UI만 있고, 실제 3개 탭은 전부 "준비 중" 플레이스홀더.
