# 작업 지시: KIS 미동작 지표 복구 (선물 수급 + 공매도 잔고)

발행: 관리자 / 수행: 개발자
브랜치: develop (main 직접 작업 금지)
우선순위: 1순위(해야 하는데 안 되어 있는 것) — 관리자 실측으로 확인

## 실측 현황 (둘 다 실패 중)
- **코스피200 선물 투자자 수급**: 근월물 코드 조회는 정상(예 `A01606`)이나
  `/uapi/domestic-futureoption/v1/quotations/inquire-futures-investor` 호출이 **404**.
  → 결과 시황 선물 수급이 항상 None + "조회 실패" 경고.
- **공매도 잔고 비율**: `short_balance_ratio`가 None.
  (공매도 거래비중 `short_sale_volume_ratio`는 1.44%로 나옴 → 일부만 동작)

## 원인
KIS 엔드포인트 path/tr_id가 실제 API와 불일치(404). 과거 종목 수급에서도
TR_ID 오류(FHKST01010300→01010900)로 같은 일이 있었음. 추정 금지, **KIS 공식 문서로 확정**.

## 요구 결과

### 1. 선물 투자자 수급 (kis_market_collector.py)
- KIS 개발자센터 "국내선물옵션" 문서에서 **투자자별 매매동향**의 정확한 path + tr_id + 요청 파라미터 + 응답 필드 확인.
- 근월물 코드(`A01606`)가 이 엔드포인트가 받는 형식이 맞는지도 확인(단축코드 vs 표준코드).
- 실제 호출로 외국인/기관 순매수 계약수가 나오게 수정.

### 2. 공매도 잔고 비율 (kis_stock_collector.py)
- 잔고 비율을 주는 정확한 엔드포인트/필드 확인. 거래비중과 별개 지표.
- KIS에 잔고 비율 엔드포인트가 없으면(거래비중만 제공) → **거래비중만 정직하게 노출하고
  잔고비율 필드는 "미제공"으로 명확히**. 추정 필드로 채우지 말 것.

## 검증 (실호출 필수 — mock 금지)
- 시황 생성 시 선물 외국인/기관 순매수에 실제 숫자가 뜨는지.
- 종목 분석에서 공매도 지표가 실값이거나, 안 되면 깔끔한 "미제공".
- 안 되는 경우 `futures_warning`처럼 사용자가 이유를 아는 상태 유지.
- `.venv/bin/python3 -m pytest` 전체 통과.
