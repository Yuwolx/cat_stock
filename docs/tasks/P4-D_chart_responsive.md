# 작업 지시: 대시보드 그래프 모바일/반응형 스크롤 버그 수정

발행: 관리자 / 수행: 개발자
브랜치: develop (main 직접 작업 금지)
근거: 주인 보고 — 폰에서 대시보드 그래프 스크롤 버그, 웹도 의심

## 진단 (관리자)
- `dashboard_styles.py`에 **plotly 차트 컨테이너용 overflow/반응형 CSS가 없음**.
- `_fig_html`는 `responsive: True`로 출력하지만, 부모 컨테이너 폭 제약/overflow 처리가 없어
  좁은 화면에서 차트가 넘치거나 터치 스크롤이 충돌할 수 있음.

## 확정 증상 (주인 확인)
아래로 잘 내려가다가 **그래프 부분부터 화면이 뜬금없이 쉭쉭 바뀌며 안 내려감**.
→ plotly 차트가 터치 드래그를 **팬/줌으로 가로채** 페이지 세로 스크롤이 막힘. (증상 B 확정)
→ 핵심 해결: 차트 인터랙션을 끈다(정적 열람용이라 줌/팬 불필요).

## 요구 결과
### 1. 차트 컨테이너 반응형 (가로 넘침 차단)
- plotly가 생성하는 `.js-plotly-plot`, `.plotly-graph-div` 및 이를 감싼 컨테이너에
  `max-width:100%; width:100%; overflow-x:hidden` 류 처리.
- 차트가 부모 폭을 절대 넘지 않게. 모바일에서 가로 스크롤 안 생기게.
- 신문 그리드(`.main-grid`, `.sidebar`)도 모바일 1열 전환 시 차트가 넘치지 않는지 확인.

### 2. 차트 인터랙션 제거 (세로 스크롤 보장 — 핵심)
- `_fig_html` plotly config를 **`staticPlot: True`** 로. (드래그/팬/줌/터치 가로채기 전부 제거)
  → 차트 위에서도 페이지가 정상 스크롤됨.
- 대시보드는 "AI에 붙여넣을 자료를 보는" 정적 열람용이라 줌/팬 불필요.
- 보강: 차트 컨테이너에 `touch-action: pan-y` 도 함께 줘서 세로 스크롤 우선.
- staticPlot이면 호버 툴팁이 사라지지만, 값은 이미 막대/텍스트로 보이므로 정보 손실 작음.
  (만약 호버를 꼭 살려야 하면 staticPlot 대신 `scrollZoom:False`+`dragmode:False`+`touch-action:pan-y`)

### 3. 코인 대시보드도 동일 점검
- 코인 대시보드에 차트 있으면 같은 처리.

## 검증
- 브라우저 좁은 폭(모바일 에뮬레이션 ~375px)에서 시황/종목 대시보드:
  가로 스크롤 없고, 세로 스크롤 정상, 차트가 폭 안에 들어오는지.
- 데스크톱 폭에서도 차트 정상 렌더(회귀 없음).
- `.venv/bin/python3 -m pytest` 전체 통과.

## 주의
- main 푸시 금지. develop 커밋만.
- 차트 데이터/종류는 그대로. 레이아웃/CSS/plotly config만.
- staticPlot 적용 시 호버 툴팁도 사라지는지 확인 — 정보 손실 크면 scrollZoom만 끄는 선에서.
