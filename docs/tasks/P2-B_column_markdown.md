# 작업 지시: AI 칼럼 마크다운 기호 노출 수정

발행: 관리자 / 수행: 개발자
브랜치: develop (main 직접 작업 금지)
근거: 주인 보고 — 대시보드 칼럼 제목에 `### `가 그대로 보임

## 문제
GPT가 제목을 `### 확산하는 반도체 기대감...`처럼 **마크다운 헤더 기호**를 붙여 반환한다.
`_column_html`은 이를 `<h2>`에 그대로 넣어 `###`가 텍스트로 노출된다.
대시보드는 마크다운 렌더가 아니라 HTML 직접 삽입이므로 기호가 안 떨어진다.

## 요구 결과

### 1. column_service.py `_split_title_body` (1차 방어, 필수)
- 제목에서 선행 마크다운 마커 제거: `^#{1,6}\s*` (예: `### `, `## `).
- 제목/본문 모두 굵게 마커(`**`, `__`)도 정리.
- 즉 GPT가 마크다운을 붙여도 화면엔 순수 텍스트만.

### 2. 프롬프트 보강 (2차 방어)
- `generate_stock_column` / `generate_market_column` 프롬프트 작성 지침에 한 줄 추가:
  "마크다운 기호(#, *, - 등) 없이 일반 텍스트로만 작성".

## 검증
- 제목이 `### ...`, `## ...`, `**...**`로 와도 대시보드에 기호 없이 제목만 크게 나오는지.
- `_split_title_body`에 마크다운 입력 단위 테스트 추가(네트워크 없이).
- `.venv/bin/python3 -m pytest` 전체 통과.
