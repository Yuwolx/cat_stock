## D. 코인 모드 정리

코인 시황은 실데이터로 잘 채워짐(4초, None 거의 없음). 아래는 잠재 리스크/미점검.

### D-1. CoinGecko rate limit 대응
- 무료 API는 분당 ~30회 제한. 반복/동시 사용 시 429 가능(기획서 리스크).
- 429 감지 시 백오프 또는 사용자 경고. 현재 무방비.
- 수용: rate limit 걸려도 앱이 깨지지 않고 사용자가 이유를 앎.

### D-2. 코인 데이터 캐싱
- 코인 서비스도 매 호출마다 외부 API를 때림(시황 4초, 24h 거의 안 변함).
- `@st.cache_data(ttl=300)` 등으로 단기 캐시. (주식 캐싱과 같은 정책으로 묶어도 됨)
- 수용: 같은 화면 반복 진입 시 외부 호출 안 함.

### D-3. 개별 코인·섹터 None 필드 전수 점검
- 시황은 점검 완료(양호). 개별 코인/섹터/학습노트 대시보드는 미점검.
- 실제 코인(예: BTC, SOL)으로 호출해 None으로 비는 지표를 찾아,
  원래 없는 값이면 "—" 표기, 수집 실패면 A/B 원칙대로 정직 처리.
- 수용: 가짜/오해 소지 값 없이, 빈 값은 빈 값으로 명확.

### D-4. Binance 펀딩비/OI · 근월물 검증
- `binance_futures_client.py`가 None 다수. 실제로 값이 들어오는지 미검증.
- 개별 코인에서 펀딩비/미결제약정이 실제 표시되는지 실호출 확인. 안 되면 정직 처리.
- 수용: 표시되는 펀딩비/OI가 실값이거나, 안 되면 깔끔히 비움.
