# 앱 데이터 흐름 가이드

작성일: 2026년 6월 3일

이 문서는 주식을 잘 모르는 사용자도 `cat_stock` 앱이 무엇을 가져오고, 그 데이터가 무슨 뜻이고, 어떤 버튼을 눌렀을 때 어떤 형태로 제공되는지 이해할 수 있게 정리한 안내서입니다.

## 1. 앱을 한 문장으로 말하면

`cat_stock`은 사용자가 버튼을 누르면 주식/코인 관련 데이터를 여러 사이트와 API에서 모아, AI에게 붙여넣기 좋은 짧은 텍스트와 사람이 보기 좋은 대시보드 HTML로 정리해주는 앱입니다.

앱이 직접 투자 판단을 대신하지는 않습니다. 역할은 "분석 재료 수집과 정리"입니다.

## 2. 출력 형태

앱은 대부분의 기능에서 같은 방식으로 결과를 제공합니다.

| 형태 | 어디서 보나 | 용도 |
| --- | --- | --- |
| 화면 출력 박스 | 오른쪽 큰 출력 영역 | 바로 읽거나 복사 |
| 복사 버튼 | 출력 박스 우상단 | ChatGPT/Gemini 프롬프트에 붙여넣기 |
| TXT 다운로드 | 생성 후 왼쪽 버튼 | 프롬프트용 원문 저장 |
| 대시보드 HTML | 생성 후 왼쪽 버튼, 미리보기 expander | 표/카드/차트로 보기 |
| output 파일 | `output/` 폴더 | 실행 결과 저장 |

중요한 구분은 이것입니다.

- TXT: AI에게 줄 핵심 지표 중심
- HTML 대시보드: 사람이 보기 좋은 카드, 표, 차트 중심
- 뉴스: TXT와 HTML 모두 제목과 원문 링크 중심

## 3. 사용자가 누르는 버튼과 데이터 흐름

### 시황 브리핑

사용자 행동:

1. STOCK 모드에서 `시황 브리핑` 탭 선택
2. `기준일` 선택
3. `브리핑 생성` 버튼 클릭

앱 내부 흐름:

`src/ui/pages.py` -> `generate_market_briefing()` -> 여러 market collector -> `format_market_briefing()` -> 화면/TXT/HTML

제공 형태:

- 오른쪽 출력 박스: 복사용 시황 텍스트
- `market_briefing_YYYYMMDD.txt`: 다운로드
- `market_dashboard_YYYYMMDD.html`: 대시보드 다운로드/미리보기
- TXT의 `주요 뉴스`와 HTML의 `LATEST NEWS`는 제목과 원문 링크를 함께 제공합니다.

### 개별 종목 분석

사용자 행동:

1. STOCK 모드에서 `개별 종목 분석` 탭 선택
2. 종목명 입력, 예: `삼성전자`
3. 리포트 기간 선택
4. `분석 생성` 버튼 클릭

앱 내부 흐름:

`src/ui/pages.py` -> `generate_stock_report()` -> stock/DART/KIS/fnguide collector -> `format_stock_report()` -> 화면/TXT/HTML

제공 형태:

- 오른쪽 출력 박스: AI에게 붙여넣기 좋은 종목 분석 텍스트
- `stock_종목명.txt`: 다운로드
- `stock_종목명_dashboard.html`: 대시보드 다운로드/미리보기
- HTML 대시보드의 최근 뉴스 카드는 원문 링크로 연결

### 테마 공부

사용자 행동:

1. STOCK 모드에서 `테마 공부` 탭 선택
2. 테마명 입력, 예: `HBM`, `2차전지`, `원전`
3. `자료 생성` 버튼 클릭

앱 내부 흐름:

`src/ui/pages.py` -> `generate_theme_report()` -> theme collector -> `format_theme_report()` -> 화면/TXT/HTML

제공 형태:

- 오른쪽 출력 박스: 테마 관련 종목/뉴스 텍스트
- `theme_테마명.txt`: 다운로드
- `theme_테마명_dashboard.html`: 대시보드 다운로드/미리보기

### CAT COIN

사용자 행동:

1. 로고 토글로 COIN 모드 전환
2. `코인 시황`, `개별 코인`, `섹터 공부`, `학습 노트` 탭에서 입력/생성

앱 내부 흐름:

각 코인 서비스가 CoinGecko, Upbit, DeFiLlama, Alternative.me, Binance futures 등에서 데이터를 가져와 텍스트와 대시보드로 정리합니다.

CAT COIN은 주식 프롬프트용보다 공부용 대시보드 성격이 강합니다.

## 4. 시황 브리핑 데이터 설명

시황 브리핑은 "오늘 한국 증시 전체 분위기"를 보는 기능입니다.

| 데이터 | 뜻 | 어디서 가져오나 | 왜 보나 |
| --- | --- | --- | --- |
| 코스피/코스닥 지수 | 한국 대표 시장의 전체 가격 흐름 | 네이버 증권 | 오늘 시장이 올랐는지 내렸는지 확인 |
| 등락률 | 어제보다 얼마나 올랐거나 내렸는지 | 네이버 증권 | 시장 강도 확인 |
| 거래대금 | 시장에서 돈이 얼마나 많이 오갔는지 | 네이버 증권 | 상승/하락에 힘이 있는지 확인 |
| 글로벌/매크로 | 미국 지수, 환율, 금리, 유가, 중국 지수 | FinanceDataReader 등 | 한국 시장에 영향을 주는 외부 분위기 확인 |
| 거래대금 상위 | 돈이 많이 몰린 종목 | 네이버 증권 거래상위 | 오늘 시장의 관심 종목 확인 |
| 업종별 등락 | 업종/그룹별 상승률 | 네이버 증권 업종 페이지 | 반도체, 자동차 같은 큰 그룹 중 어디가 강한지 확인 |
| 외국인/기관/개인 수급 | 투자자별 순매수/순매도 | 네이버 증권/KRX 계열 수집 | 누가 사고 누가 파는지 확인 |
| 프로그램 매매 | 자동/기관성 매매 흐름 | 네이버 증권 | 수급이 시장을 밀고 있는지 확인 |
| 상한가 | 하루 제한폭까지 오른 종목 | 네이버 증권 상한가 페이지 | 과열/테마 급등 종목 확인 |
| 5% 이상 상승 종목 | 등락률 상위 중 +5% 이상 종목 | 네이버 증권 상승 페이지 | 강하게 움직인 종목을 빠르게 추림 |
| 시간외 단일가 급등락 | 장 마감 후 움직인 종목 | 네이버 증권 NXT/시간외 페이지 | 다음 날 이슈 후보 확인 |
| 주요 뉴스 | 시장 주요 기사 제목과 원문 링크 | 네이버 증권 주요뉴스 | 시장에서 주목하는 이슈 확인 |

초보자 기준으로는 이렇게 보면 됩니다.

- 지수: 시장 전체 온도
- 거래대금: 돈의 세기
- 수급: 누가 사고 파는지
- 상승 종목/상한가: 오늘 시장의 이슈
- 뉴스: 시장에서 지금 말이 나오는 사건

CAT COIN은 "코인을 공부하기 위한 대시보드"입니다. 주식 모드처럼 프롬프트용 텍스트도 만들지만, 화면에서 시장 구조를 이해하는 쪽에 더 가깝습니다.

| 기능 | 데이터 | 출처 | 의미 |
| --- | --- | --- | --- |

## 8. 환경변수와 데이터 실패

일부 데이터는 API 키가 있어야 제대로 들어옵니다.

| 키 | 쓰는 곳 | 없거나 틀리면 |
| --- | --- | --- |
| `DART_API_KEY` | 공시, 재무, 대주주 지분율 | DART 항목이 더미/실패 메시지/빈 결과로 표시 |
| `KIS_APP_KEY`, `KIS_APP_SECRET` | KIS 수급 금액, 공매도 거래량 비중, 선물 수급 보강 | 해당 항목이 None 또는 네이버 기반 대체값으로 표시 |
| `OPENAI_API_KEY` | 시황/종목 AI 칼럼 생성 | TXT/대시보드는 생성되지만 AI 칼럼은 표시되지 않음 |

DART는 OpenDART에서 발급받은 40자리 인증키만 넣어야 합니다. `.env`에는 URL이나 `crtfc_key=` 같은 파라미터를 같이 넣으면 안 됩니다.

정상 예시:

```env
DART_API_KEY=40자리인증키만
```

## 9. 왜 어떤 날은 데이터가 없어 보이나

데이터가 없다고 해서 항상 오류는 아닙니다.

- DART 당일 공시: 새벽이나 장 전에는 당일 공시가 없을 수 있습니다.
- 네이버 페이지: 페이지 구조가 바뀌면 일부 파싱이 깨질 수 있습니다.
- 글로벌 지표: 휴장일/데이터 제공 지연이면 비어 있을 수 있습니다.
- KIS 데이터: API 키가 없으면 보강 데이터가 비어 있을 수 있습니다.
- FinanceDataReader: 종목명 검색이 안 되면 기본 시세가 비어 있을 수 있습니다.

앱은 실패한 항목을 가능한 한 화면 경고나 "데이터 없음"으로 보여주도록 되어 있습니다.

## 10. 초보자용 읽는 순서

처음에는 모든 항목을 한 번에 이해하려 하지 않아도 됩니다.

시황 브리핑은 이 순서로 봅니다.

1. 코스피/코스닥이 올랐나 내렸나
2. 거래대금이 큰가 작은가
3. 외국인/기관이 샀나 팔았나
4. 어떤 업종과 종목이 강했나
5. DART 공시나 뉴스성 이벤트가 있었나

개별 종목은 이 순서로 봅니다.

1. 가격과 등락률
2. 거래대금과 수급
3. 재무와 밸류에이션
4. 뉴스/공시/리포트
5. 위험 키워드

이 앱의 출력 텍스트는 "정답"이 아니라 "AI에게 줄 좋은 재료"입니다. 최종 판단은 추가 확인과 본인의 기준이 필요합니다.
